# Hybrid Quantum Control Scheduler (SimPy + QuTiP)

It simulates:
- **Classical traffic** (packets/jobs) competing with
- **Quantum workload** that consumes *quantum credits* (e.g., entanglement pairs / QKD key chunks)

Two scheduling policies are included:
1. **Time-gated (TSN-like)**: periodic gate windows for quantum vs classical.
2. **Credit/event-driven**: prioritize quantum when the credit buffer is full (or when credits exist).

A lightweight **quantum link emulator** uses **QuTiP** to model a noisy Bell-pair generation attempt
and turns successful attempts into credits.


## Quickstart (local)

```bash
python -m venv .venv
# Windows: .venv\Scripts\activate
source .venv/bin/activate

pip install -U pip
pip install -e ".[dev]"

# Run a scenario
hqcsim run --policy credit --t-end 5 --classical-lam 800 --quantum-lam 120 --credits-capacity 200
```

Outputs are printed as JSON summary.

## Quickstart (Docker)

```bash
cd infra
docker compose up --build
```

Then run inside container (see container logs for shell tip), or use local install for CLI.

## CLI Examples

```bash
hqcsim run --policy credit --t-end 5 --classical-lam 800 --quantum-lam 120 --credits-capacity 200 --stochastic-credit-rate 200

```

## Notes
- If QuTip is not desired for simulation, `--qutip` can be simply dropped(credits can be generated by a simple stochastic model).
- This is a **control software/scheduling** demonstration; it is not a full physical trapped-ion simulator.
